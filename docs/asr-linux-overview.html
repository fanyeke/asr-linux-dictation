ASR Linux

AI 驱动的 Linux 桌面语音输入工具。按热键，说话——你的话经过云端 ASR 识别、LLM 润色后，精准出现在当前聚焦的窗口里。
01 这是什么？

ASR Linux 是一款为 Linux 桌面打造的 AI 语音输入 应用。它填补了 Linux 桌面生态中长期缺失的一环——像 Windows 上的语音输入那样，按下热键就能说话输入，而且利用 LLM 做润色，去掉口头禅、加标点、修正识别错误。

整个流程是：热键触发 → 录制音频 → 云端 ASR 识别 → LLM 润色 → 智能词典替换 → 注入当前窗口。全程 隐私优先，后端只绑定 127.0.0.1，API 密钥走 Linux Secret Service，日志自动脱敏。

06 工作流详解

一次完整口述的全流程：

┌─ 用户按热键 ───────────────────────────────┐
│                                              │
│  ① AudioRecorder.start() → arecord pipe     │
│     ● 同时 fire-and-forget 预热 ASR/LLM 连接  │
│     ● RingBuffer 累积 PCM 数据（流式 ASR 用）  │
│     ● 可选 VAD 静音检测，overlay 显示倒计时    │
│                                              │
├─ 用户再次按热键 / VAD 自动停止 ──────────────┤
│                                              │
│  ② ASRClient.transcribe() → Mimo API         │
│     ● base64 编码 WAV → multipart 请求       │
│     ● 指数退避重试（timeout/429/5xx）        │
│     ● 输出：原始文本                         │
│                                              │
│  ③ DictionaryManager 查询                    │
│     ● 按当前场景配置过滤词典条目              │
│     ● LLM prompt 注入词典映射                │
│                                              │
│  ④ PolishClient.polish() → LLM               │
│     ● prompt_template + 原始文本 + 词典      │
│     ● 去掉"嗯""那个"，加标点，修正识别        │
│     ● 默认模型：opencode-go/deepseek-v4-flash │
│                                              │
│  ⑤ TextInjector.inject() → 聚焦窗口          │
│     ● 保存当前剪贴板内容                     │
│     ● xdotool type 注入 / 剪贴板粘贴         │
│     ● 成功 → 恢复剪贴板；失败 → 保留文本    │
│                                              │
│  ⑥ HistoryStore 写入 + 音频清理              │
│     ● 成功 → 删录音文件                      │
│     ● 失败 → 保留录音 + 记录 error_type      │
│                                              │
└──────────────────────────────────────────────┘

关于流式 ASR (Phase 13 🚧)：目前是"录完再识别"的批处理模式。正在进行流式改造——录音期间后台切片（3s 切片，1s 重叠）并发 ASR，实时广播部分结果到 overlay 预览区，结束时长后缀合并送 LLM 润色。

09 设计决策
决策 选择 理由
录音方式 arecord pipe mode 零额外依赖，ALSA 内建，pipe 支持流式处理
ASR 提供商 Mimo API（OpenAI 兼容） 可替换，环境变量切换 base_url
默认 LLM opencode-go/deepseek-v4-flash 快且便宜，可替换
文本注入 xdotool type + 剪贴板回退 兼容性好，Terminal 也可用
API 密钥 Linux Secret Service 系统级加密存储，不在配置文件明文
词典匹配 全量传入 LLM，不做本地模糊匹配 简化实现，LLM 上下文理解更准确
场景管理 DB 多场景配置 + Tray 切换 用户按场景切换 prompt/词典，无需反复配置
架构边界 Electron 不碰 ASR/润色逻辑 模块清晰，后端可独立测试
